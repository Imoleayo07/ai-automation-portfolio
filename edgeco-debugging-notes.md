# Edgeco Debugging Notes

A log of every bug hit while getting the Edgeco job search automation working end to end (Antigravity → proxy.py → n8n cloud → Firecrawl → AI agent → Supabase-style portal).

## 1. WinError 10061 (Connection Refused)

**Symptom:** `proxy.py` returned `502 Bad Gateway` with `<urlopen error [WinError 10061]>` even though direct PowerShell requests to the same n8n webhook returned `200 OK`.

**Root cause:** A stale `proxy.py` process was still running in the background from an earlier test, serving outdated code that pointed at the wrong webhook URL. Stopping the server with `Ctrl+C` in one terminal didn't kill an older process started in a different tab.

**Fix:**
```
taskkill /F /IM python.exe
python proxy.py
```
Always confirm only one instance is running before testing.

**Lesson:** when local code changes don't seem to take effect, check for zombie processes before assuming the code is wrong.

## 2. Cloudflare 524 Timeout

**Symptom:** Portal showed `Server error (524): error code: 524` after the WinError was fixed.

**Root cause:** n8n cloud sits behind Cloudflare, which kills any request after 100 seconds. The AI agent's scrape-and-extract loop was taking longer than that.

**Fix:** Reduce job boards per search, trim scraped content to main page text only, use a faster model, lower max agent iterations. (Long-term fix if still too slow: switch to async pattern — webhook responds immediately, workflow finishes in background, portal polls for results.)

## 3. AI Agent Max Iterations Reached

**Symptom:** `Max iterations (10) reached. The agent could not complete the task.`

**Root cause:** Underlying Firecrawl call was failing (402 Payment Required), so the agent kept retrying instead of finishing.

## 4. Firecrawl 402 — Insufficient Credits

**Symptom:** Tool call failed with `Request failed with status code 402`.

**Root cause:** Confusingly, the account actually had 1,391 credits remaining — turned out to be a red herring on this attempt; underlying issue was schema/key mismatch, not actual balance.

**Fix:** Verified API key in n8n credential matched the Firecrawl dashboard key exactly.

## 5. Tool Schema Mismatches (multiple models, multiple errors)

**Symptoms across several runs:**
- `missing properties: 'url'`
- `additionalProperties 'scrape_options', 'job_filter', 'query' not allowed`
- `additionalProperties 'url', 'formats', 'jsonOptions', 'onlyMainContent', 'waitFor' not allowed`

**Root cause:** Multiple LLMs (Llama 3.3 70B, GPT-OSS) defaulted to Firecrawl's *public REST API* field names instead of the actual schema generated by the `@mendable/n8n-nodes-firecrawl` community node, which exposes a completely different set of fields: `url`, `parsers`, `scrapeOptions`, `profile`, `useCustomBody`.

**Fix:** Opened the Firecrawl node directly in n8n to read its real exposed parameters instead of guessing from documentation. Added an explicit system prompt instruction listing only the valid field names.

**Lesson:** when an agent's tool calls keep failing in different ways, stop guessing and go read the actual node's parameter list. Models will substitute memorized API shapes for the real schema if not told otherwise.

## 6. "No parameters set up to be filled by AI"

**Root cause:** The URL field in the Firecrawl node wasn't toggled to "From AI" — the agent had no way to actually pass in a URL, so every call failed against an effectively empty schema.

**Fix:** Clicked the sparkle (✨) icon next to the URL field to enable AI control. Left other fields (Parsers, Scrape Options) on fixed/manual values to reduce complexity.

## 7. Community Node Bug — `actions` Field

**Symptom:** `Request failed with status code 400` — `expected: array, received: object` for a field called `actions`.

**Root cause:** A bug in the `@mendable/n8n-nodes-firecrawl` node itself; it auto-generates a malformed request body.

**Fix:** Enabled "Use Custom Body" on the node to bypass the broken auto-generated payload and send a minimal hand-written JSON body instead.

## 8. Groq Free Tier Token Limit

**Symptom:** `Request too large for model llama-3.3-70b-versatile ... Limit 12000, Requested 17639`.

**Root cause:** Scraped page content (including nav/footer/ads, not just job listings) pushed the prompt over Groq's free-tier 12k tokens-per-minute cap.

**Fix:** Switched to a smaller model (`llama-3.1-8b-instant`) and trimmed Firecrawl's scrape output to main content only.

## 9. Agent Returning Prose Instead of JSON

**Symptom:** `Parse Companies` node crashed with `Invalid JSON - Unexpected token 'I', "I searched"...`.

**Root cause:** No explicit instruction forcing pure JSON output, so the agent explained its reasoning in plain English instead.

**Fix:** Added a hard constraint to the system prompt: final response must be ONLY valid JSON matching `{"companies": [...]}`, no commentary, no markdown.

## 10. Empty Results Treated as a Hard Error

**Symptom:** `Agent returned 0 companies` halted the whole workflow, and the portal showed a red error screen for a legitimate "no matches" result.

**Fix (n8n):** Changed `Parse Companies` to let an empty array pass through instead of throwing.

**Fix (frontend):** Updated `script.js` to stop treating zero jobs as an error — `renderJobs()` now shows a "No matching jobs found" message inside the normal completed state instead of triggering the error panel.

---

## Summary of Recurring Themes

1. **Stale processes** can make fixed code look broken — always confirm a clean restart.
2. **Infra timeouts (Cloudflare's 100s cap)** are invisible until you hit them — design for them if scrape/AI steps are slow.
3. **Models override given schemas with memorized API patterns** — always verify the actual node's exposed parameters instead of trusting documentation or the model's assumptions.
4. **Free-tier rate limits** are real constraints on production reliability, not just a vague "use a cheaper model" suggestion — match model size to token budget.
5. **Force structured output explicitly** — agents default to conversational prose unless told not to.
6. **Treat "zero results" as a valid outcome, not a failure** — both in workflow logic and frontend UX.
